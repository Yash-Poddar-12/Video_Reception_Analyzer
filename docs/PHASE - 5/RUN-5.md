## 🔷 Part 3: Power BI Integration (Optional)

**⚠️ IMPORTANT**: Power BI Desktop only works on Windows. macOS users need:
- Windows VM (Parallels, VMware Fusion)
- Boot Camp partition
- Separate Windows machine
- **OR skip Power BI** and use static visualizations only

### Step 1: Install Power BI Desktop (Windows)

1. Download: https://powerbi.microsoft.com/desktop
2. Run installer: `PBIDesktopSetup_x64.exe`
3. Launch Power BI Desktop
4. Sign in with Microsoft account (free)

### Step 2: Import Training Data

1. **Get CSV → From Text/CSV**
2. Browse to: `data/processed/reviews_clean.csv`
3. Click **Load** (not Transform)
4. Verify data appears in Fields pane

**Expected columns:**
- text (string)
- sentiment (string)
- word_count (number)
- char_count (number)

### Step 3: Enable R Visuals

1. **File → Options and settings → Options**
2. Navigate to: **R scripting**
3. Set **R installation directory**:
   - Windows: `C:\Program Files\R\R-4.x.x\bin\x64`
   - Check with: `where R` in Command Prompt
4. Click **OK**

### Step 4: Create R Visuals

Use the code snippets in the next section to create each visualization.

**For each visual:**

1. Click **R script visual** icon (📊 with R)
2. Drag required fields to **Values** section
3. Paste R code into script editor
4. Click **Run script** (▶️ button)
5. Visual appears in report canvas
6. Resize and position as desired

**Recommended layout:**
- 3x3 grid (9 visuals total)
- Sentiment Distribution (top-left, largest)
- Charts arranged by importance

### Step 5: Publish to Power BI Service

1. **Click "Publish" button** (top ribbon)
2. Select destination workspace (default: "My workspace")
3. Click **Select**
4. Wait for upload to complete
5. Click **Open '[Report Name]' in Power BI**

### Step 6: Get Embed URL

**In Power BI Service (web browser):**

1. Navigate to your published report
2. Click **File → Embed report → Publish to web (public)**
   - ⚠️ **CAUTION**: This makes your dashboard publicly accessible!
   - For production, use "Embed in SharePoint Online" or "Secure embed"
3. Click **Create embed code**
4. Click **Publish** to confirm
5. Copy the **Link** (not the iframe code)
   - Format: `https://app.powerbi.com/view?r=XXXXX`

**For private embedding:**
- Use "Embed for your organization" instead
- Requires Azure AD authentication
- See: https://docs.microsoft.com/power-bi/developer/embedded/

### Step 7: Configure Frontend

1. **Open `frontend/.env.local`:**
   ```bash
   nano frontend/.env.local
   # or use your preferred editor
   ```

2. **Add embed URL:**
   ```env
   NEXT_PUBLIC_POWERBI_EMBED_URL=https://app.powerbi.com/view?r=XXXXX
   ```

3. **Restart frontend:**
   ```bash
   cd frontend
   # Press Ctrl+C to stop
   npm run dev
   ```

4. **Test:**
   - Go to: http://localhost:3000/analytics
   - Should see iframe with your Power BI dashboard
   - Interact with filters and drill-downs

## 📝 Power BI R Code Snippets

Copy these into Power BI's R visual script editor.

### 1. Sentiment Distribution

**Required Fields:**
- sentiment

```r
library(ggplot2)
library(dplyr)

# Aggregate sentiment counts
sentiment_counts <- dataset %>%
  group_by(sentiment) %>%
  summarise(count = n())

# Color mapping
colors <- c(
  "positive" = "#22c55e",
  "negative" = "#ef4444",
  "neutral" = "#f59e0b"
)

# Create bar chart
ggplot(sentiment_counts, aes(x = sentiment, y = count, fill = sentiment)) +
  geom_bar(stat = "identity", width = 0.7) +
  geom_text(aes(label = count), vjust = -0.5, size = 5) +
  scale_fill_manual(values = colors) +
  labs(
    title = "Sentiment Distribution",
    x = "Sentiment",
    y = "Count"
  ) +
  theme_minimal() +
  theme(
    plot.title = element_text(size = 16, face = "bold", hjust = 0.5),
    legend.position = "none",
    axis.text = element_text(size = 12)
  )
```

### 6. Sentiment Pie Chart

## 🔧 Troubleshooting

### Visualizations Not Appearing

**Problem:** Analytics page shows "Visualization not found"

**Solution:**
```bash
# 1. Check if files exist
ls -la visuals/

# 2. Check symlink
ls -la frontend/public/visuals

# 3. If symlink broken, recreate
cd frontend/public
rm visuals  # Remove old symlink
ln -sf ../../visuals visuals

# 4. Restart frontend
cd ..
npm run dev
```

### R Script Fails

**Problem:** `bash setup_phase5.sh` errors out

**Solutions:**

**Error: "training data not found"**
```bash
# Create sample data
mkdir -p data/processed
cat > data/processed/reviews_clean.csv << 'EOF'
text,sentiment,word_count,char_count
"Great movie",positive,2,12
"Bad film",negative,2,8
EOF
```

**Error: "package 'ggplot2' is not available"**
```bash
# Install R packages manually
R
> install.packages("ggplot2")
> # Repeat for other packages
> quit()
```

**Error: "object 'char_count' not found"**
- Your CSV is missing required columns
- Add: `word_count,char_count` columns
- See sample data format above

### Power BI Issues

**Problem:** R visuals not working in Power BI

**Solutions:**

1. **Enable R scripting** (Options → R scripting)
2. **Install R packages in Windows R:**
   ```r
   install.packages(c("ggplot2", "dplyr", "reshape2"))
   ```
3. **Check R path** is correct in Power BI options
4. **Use Power BI's R console** for testing:
   - Paste code
   - Check for errors

**Problem:** Embed URL not working

**Solutions:**

1. **Check format:**
   - Should be: `https://app.powerbi.com/view?r=XXXXX`
   - Not iframe embed code

2. **Restart frontend** after adding env variable

3. **Check browser console** for iframe errors

4. **Try "Publish to web"** instead of private embed for testing

### Frontend Build Errors

**Problem:** `next build` fails

**Solutions:**

```bash
# Clear Next.js cache
cd frontend
rm -rf .next

# Reinstall dependencies
rm -rf node_modules package-lock.json
npm install --legacy-peer-deps

# Build again
npm run build
```

### Image Loading Issues

**Problem:** Next.js Image component errors

**Solution:**

Add to `next.config.js`:
```javascript
const nextConfig = {
  images: {
    domains: ['localhost'],
    unoptimized: true, // For static export
  },
};
```

## ❓ FAQ

### Do I need Power BI?

**No!** Power BI is completely optional. The analytics page works perfectly with just the R-generated static visualizations.

**Use static visualizations if:**
- You're on macOS and don't have Windows VM
- You don't need interactive filtering
- You want a simple setup
- You're prototyping or developing

**Use Power BI if:**
- You have Windows available
- You want interactive drill-down
- You need to share dashboards with stakeholders
- You want real-time data refresh

### Can I use Power BI on macOS?

Power BI Desktop is Windows-only. Options:

1. **Parallels Desktop** (paid) - Run Windows VM
2. **VMware Fusion** (paid/free) - Run Windows VM  
3. **Boot Camp** (free, Intel Macs only) - Dual boot Windows
4. **Separate Windows PC** - Use different machine
5. **Skip Power BI** - Use static visualizations only

### How do I update visualizations?

**After retraining models or getting new data:**

```bash
# 1. Retrain models (if needed)
cd r
Rscript train_model.R

# 2. Regenerate visualizations
cd ..
Rscript r/eda.R

# 3. Refresh Power BI (if using)
# In Power BI Desktop: Home → Refresh
# Then republish to Power BI Service
```

### Can I customize the visualizations?

**Yes!** Edit `r/eda.R`:

```bash
# Open in editor
code r/eda.R  # VS Code
# or
nano r/eda.R
```

**Change colors:**
```r
# Around line 49
COLORS <- list(
    positive = "#YOUR_COLOR",
    negative = "#YOUR_COLOR",
    neutral = "#YOUR_COLOR"
)
```

**Change resolution:**
```r
# In each ggsave() call
ggsave("visuals/filename.png", dpi = 600, ...)  # Higher res
```

**Add new visualizations:**
1. Add code to `r/eda.R`
2. Save PNG to `visuals/`
3. Add to `frontend/src/app/(protected)/analytics/page.tsx`
4. Add to VISUALIZATIONS array

### How do I secure Power BI embedding?

The "Publish to web" method makes dashboards **publicly accessible**.

**For production:**

1. **Use "Embed for your organization"**
   - Requires Azure AD authentication
   - Users must have Power BI Pro license
   - More secure, user-based access control

2. **Use "Secure embed token"**
   - Programmatic embedding
   - Row-level security (RLS) support
   - Requires backend to generate tokens
   - See: https://docs.microsoft.com/power-bi/developer/embedded/

3. **Use Power BI Embedded**
   - Azure-based embedding solution
   - Full API control
   - Pay-per-use pricing
   - Most flexible but more complex

### What if I want more visualizations?

**Static visualizations:**
1. Edit `r/eda.R` - add more `ggplot2` charts
2. Run `Rscript r/eda.R`
3. Update `analytics/page.tsx` VISUALIZATIONS array

**Power BI:**
1. Add more R visual tiles to your report
2. Paste code from this document
3. Republish to Power BI Service

### How do I deploy this to production?

**Frontend (Vercel/Netlify):**
1. Push code to GitHub
2. Connect repo to Vercel
3. Add environment variables in Vercel dashboard
4. Deploy

**Important:** 
- Symlink (`frontend/public/visuals`) won't work in production
- Copy `visuals/*.png` to `frontend/public/visuals/` instead
- Or use cloud storage (S3, Cloudinary) and update image URLs

**Backend:**
- See RUN-4.md for backend deployment
- Consider using cloud R execution (AWS Lambda with R, Google Cloud Run)
